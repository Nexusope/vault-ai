# Database Design

The runnable edge build uses SQLite-compatible Cloudflare D1 through Drizzle. Four tables ship now: `ideas`, `collections`, `imports`, and `notifications`. `ideas.id` is a UUID text primary key; source and source URL retain provenance; title, summary, creator, media type, transcript, JSON tags, trend score, confidence, status, collection id, and timestamps support the visible workflows. Indexes cover newest-first and trend-score queries. `ensureSchema()` performs idempotent local initialization while checked-in migrations govern hosted changes.

The enterprise PostgreSQL model expands to `users`, `organizations`, `memberships`, `workspaces`, `sources`, `ideas`, `media_assets`, `transcripts`, `analysis_runs`, `embeddings`, `graph_nodes`, `graph_edges`, `collections`, `collection_items`, `fusion_sessions`, `fusion_sources`, `fusion_outputs`, `trends`, `recommendations`, `notifications`, `jobs`, `provider_calls`, `audit_events`, and `outbox_events`. Every tenant-owned table includes `organization_id`, immutable UUID, `created_at`, `updated_at`, and optional `deleted_at`. Composite indexes begin with `organization_id` to prevent cross-tenant scans.

Key invariants: `(organization_id, source_id, external_id)` is unique; a media asset has exactly one workspace owner; graph edges reference existing nodes and use a canonical ordered pair for symmetric relations; prompt/provider calls never store raw secrets; deletions enqueue media/vector cleanup; and outbox records commit in the same transaction as domain changes.

Migrations are forward-only, reviewed SQL. Destructive changes use expand/migrate/contract: add nullable field, backfill in batches, dual-read, enforce constraint, then remove the old field in a later release. Backups run daily with 30-day retention and quarterly restore drills. Production RPO is 15 minutes and RTO is 60 minutes. Sensitive columns use application-layer envelope encryption where provider-managed disk encryption is insufficient.
